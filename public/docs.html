Documentation

Everything you need to install, deploy and master Cloud Panel — the open-source game server management panel.

Introduction

Cloud Panel is a self-hosted, single-node control panel for game servers — it runs everything on one host (no remote daemon). It gives you a real-time console, true CPU/RAM/disk metrics, a built-in file manager, per-server SFTP, one-click server installs ("eggs"), an admin area for locations & IP:port allocations, and a companion Discord bot — all wrapped in a fast, beautiful UI.

  • Web panel & API on port 8080.
  • SFTP on port 5657.
  • Durable SQLite database with an automatic JSON fallback.
  • No default passwords — you create your admin on first run.
Single-node by design. Servers run on the same host as the panel. The Nodes / Locations / Allocations admin screens organize ports and metadata, but there is no separate per-node daemon, so you can't yet distribute servers across multiple machines. Plan capacity for one box.
New here? Jump to the Quick Start to try it locally, or the VPS Install to go live in one command.

Requirements

  • Node.js 18+ (20+ recommended) — required for the panel.
  • Java (Temurin 21) — only if you run Minecraft servers. The VPS installer adds it automatically.
  • OS: Linux (Ubuntu/Debian recommended for the installer), or Windows/macOS for local use.
  • Open ports 8080 (web) and 5657 (SFTP), plus your game port range (default 25565-25600).

Quick Start (local)

Try Cloud Panel on your own machine in under a minute.

bashCopy
cd cloud-panel
npm install
npm start

Open http://localhost:8080 and the first-run wizard will walk you through creating your administrator. That's it — create a server and hit Start.

The Demo egg boots instantly with no Java required — perfect for trying the console and live stats.

VPS Install (one command)

On a fresh Ubuntu/Debian server, upload or clone the project, then run the installer:

your-vps — bashCopy
git clone https://github.com/Notbangbang-dev/cloud-panel.git cloud-panel
cd cloud-panel
sudo bash scripts/install.sh

The installer will:

  • Install Node.js, build tools and Java (Temurin).
  • Create a locked-down cloudpanel user and install to /opt/cloud-panel.
  • Generate a .env with a random JWT secret & your public IP.
  • Create your administrator (prompts, or auto-generates a strong password and prints it).
  • Register a systemd service and open the firewall.

Managing the service

bashCopy
systemctl status cloud-panel      # service state
journalctl -u cloud-panel -f      # live logs
sudo cloud-panel-update           # update to the latest version
sudo bash scripts/uninstall.sh    # remove (--purge to delete data)

Updating

One command pulls the latest code, keeps your .env, database & backups, reinstalls dependencies and restarts the service — no need to re-run the installer:

bashCopy
sudo cloud-panel-update

First time only — installs the global cloud-panel-update command (works on any existing install):

bashCopy
curl -fsSL https://raw.githubusercontent.com/Notbangbang-dev/cloud-panel/main/scripts/update.sh | sudo bash
After that, just run sudo cloud-panel-update anytime — it's a permanent command available to every user (with sudo).
For a public domain, put Cloud Panel behind Nginx/Caddy with TLS, proxying http://127.0.0.1:8080. Keep port 5657 open for SFTP.

First-run Setup

Cloud Panel ships with no accounts. Create your first administrator one of two ways:

Web wizard

Just open the panel — if no users exist you'll see the setup wizard. Enter a username, email and password and you're in.

CLI

bashCopy
npm run setup
# or non-interactively:
node src/scripts/setup.js --username admin --email you@example.com --password 'S3cret!!'

Servers & Eggs

An egg is a server template. When you create a server and pick an egg, Cloud Panel resolves the version, downloads the real files and (for Minecraft) accepts the EULA.

Create a server

  • Go to Admin → Servers → Create Server.
  • Choose an owner, node and egg, set memory/disk/CPU, and edit the egg's variables (version, build…).
  • Auto-installing eggs provision in the background — watch progress live in the console.

Included eggs (27)

  • Minecraft: Java — Paper, Purpur, Folia, Fabric, Quilt, Pufferfish, Leaf, Vanilla, Forge, NeoForge and Sponge (all auto-download/install). Spigot is upload-your-jar — use Paper for auto-install with plugin support.
  • Modpacks — one-click Modrinth modpack: enter a pack and Cloud Panel installs every mod, override and the matching loader.
  • Proxies — Velocity, Waterfall, BungeeCord (auto-download) and Geyser (Bedrock → Java bridge).
  • Other games — Minecraft: Bedrock, PocketMine-MP and Terraria, plus SteamCMD games (Rust, Valheim, Counter-Strike 2).
  • Generic — any Java jar, Node.js app, Python app.
  • Demo — a simulated server that needs nothing but Node.
Need a different game? Add an entry to the egg catalog (and an installer function if it should auto-download). It appears in the picker on next start.

Teams, security & databases

  • Subusers — invite other accounts to a server (server's Subusers tab) with granular permissions: console, commands, power, files, backups, schedules, databases, players, startup, network, settings.
  • Scheduled tasks — the Schedules tab runs commands, power actions or backups on a 5-field cron (e.g. 0 5 * * * for a 5am restart).
  • Two-factor auth — enable TOTP under Account; scan the QR with any authenticator app and save your recovery codes.
  • Databases — add a MySQL/MariaDB host in Admin → Databases (needs the optional mysql2 package), then create per-server databases from the server's Databases tab.
  • Plugins/mods — search Modrinth and one-click install from the Mods tab.
  • Players, metrics & status — a live Players list (kick/ban), historical Metrics graphs, and an opt-in public status page at /status/<slug>.
  • Isolation & sandboxing — opt-in OCI container sandbox (CP_OCI=1 with Docker/Podman) runs each server in its own locked-down container; or use per-server-user isolation. See SECURITY.md.

Files & SFTP

Edit files in the browser from a server's Files tab, or connect any SFTP client on port 5657.

FieldValue
Hostyour panel's IP / domain
Port5657
Username<your-username>.<server-id> (shown on the Network tab)
Passwordyour account password

Each login is scoped to that one server's files. Works great with FileZilla, WinSCP and the VS Code SFTP extension.

Users & CLI

Admins can manage users under Admin → Users. From the terminal:

bashCopy
npm run user list
npm run user create --username bob --email b@x.io --password 'pw' --admin
npm run user passwd bob
npm run user promote bob      # grant admin
npm run user demote bob       # revoke admin
npm run user delete bob --yes
Safety built in: the CLI won't delete or demote the last administrator, and won't delete a user who still owns servers.

REST API

The panel exposes a JSON API under /api. Authenticate by logging in to get a JWT, then send it as Authorization: Bearer <token>.

get a tokenCopy
curl -X POST http://localhost:8080/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"login":"admin","password":"your-pw"}'
MethodEndpointDescription
GET/api/setup/statusWhether first-run setup is needed
POST/api/setupCreate the first admin (only when none exist)
POST/api/auth/loginGet a JWT
GET/api/serversList your servers (with live resources)
GET/api/servers/:id/resourcesLive status, CPU, RAM, disk
POST/api/servers/:id/powerstart · stop · restart · kill
POST/api/servers/:id/commandSend a console command
GET/api/servers/:id/files/listList files in a path
POST/api/admin/serversCreate a server (admin)

A live console stream is available over WebSocket at /api/servers/:id/ws?token=<jwt>.

FAQ

Is it free?

Yes — MIT licensed and fully self-hostable.

Does it use Docker?

By default servers run as host processes (CPU/RAM/disk are real). For full isolation, enable the built-in OCI container sandbox — set CP_OCI=1 with Docker or Podman installed and every server runs in its own locked-down container (dropped capabilities, no-new-privileges, a PID cap and hard RAM/CPU limits). It's opt-in; see SECURITY.md.

Where is data stored?

In data/cloud-panel.db (SQLite). Per-server files live under data/volumes/<id>.
